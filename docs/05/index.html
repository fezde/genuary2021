#Genuary2021

January 5th

Prompt

Do some code golf! How little code can you write to make something interesting? Share the sketch and its code together if you can.

Interpretation

"As little code" in Plingo translates as "as small an input picture as possible".
The smallest possible image would have been a one-pixel-image. But the possibilities there are quite limited (you can go for add, substract, or invert as commands), so I decided to do a second, little more complex program.
This program consists of a 4 by 3 pixels. All but the top left pixels are initialized so they call noop and thus do not change. The top left pixel is initialized with RGB(13, 1, 1) which translates as swap_plus_plus(1,1) . This will make the first pixel "travel" one pixel down and to the right, switching the place with the pixel that occupied that space before. This makes the pixel travel across the canvas while shuffling the rest of the image.
This video was created using this image as an input. The video presented here has been scaled up a lot to make the code and its result better viewable.
The execution is basically (without the video creation): plingo -i 1000 05-golf.png
